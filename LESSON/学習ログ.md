# STM32 Nucleo-F103RB 学習メモ

## 1. 今回やったことの全体像

今回やったことは、以下の流れです。

1. **STM32CubeMX でピン設定を行う**
2. **STM32CubeIDE でコードを書く**
3. **GPIO入力でボタン状態を読む**
4. **GPIO出力でLEDを点灯・消灯する**
5. **UARTでシリアル出力する**
6. **配線とソフト設定が一致しているか確認する**

---

## 2. 開発の役割分担

### STM32CubeMX の役割
STM32CubeMX は、**マイコンの初期設定をGUIで行うツール**。

主にやること：

- 使用するマイコンやボードを選ぶ
- ピンに役割を割り当てる
- GPIO入力/出力を設定する
- UARTなどの周辺機能を有効にする
- 初期化コードを自動生成する

### STM32CubeIDE の役割
STM32CubeIDE は、**コードを書く・ビルドする・書き込むための統合開発環境**。

主にやること：

- CubeMXが生成したプロジェクトを開く
- `main.c` に自分の処理を書く
- ビルドする
- マイコンへ書き込む
- デバッグする

### 使い分けの考え方
- **設定は CubeMX**
- **処理の中身は CubeIDE**

この分担で考えると整理しやすい。

---

## 3. 今回の目標回路

今回作った回路の目的は以下。

- **外付けボタンを押したらLEDを点灯**
- **必要ならUARTで文字を送信**

### 配線の考え方

#### LED
- マイコンの出力ピン → 抵抗 → LED → GND

#### ボタン
- マイコンの入力ピン → ボタン → GND
- 入力ピンは **Pull-up** に設定

### Pull-up の意味
入力ピンを何もしないときに内部的に High にしておく設定。

そのため今回のボタン入力はこうなる。

- 押していない → `High`
- 押した → `Low`（GNDに落ちる）

つまり、**押したときは `GPIO_PIN_RESET`** と判定する。

---

## 4. 超重要：GND共通

今回一番大事だったこと：

**マイコンとブレッドボードの GND は必ずつなぐ。**

### なぜ必要か
電気は、出るだけでは流れない。  
**戻る道（GND）が必要**。

#### LEDの場合
`出力ピン → 抵抗 → LED → GND`
という閉じた経路になって初めて光る。

#### ボタンの場合
Pull-up入力では、押したときにGNDへ落として Low を作る。  
しかし、GNDが共通でないと「どこを基準にLowか」が決まらない。

### 今後の教訓
動かないときは最初に以下を確認する。

1. GND共通か
2. 電源経路が閉じているか
3. ピン設定と配線が一致しているか

---

## 5. STM32CubeMX でのピン設定

## 5.1 今回の設定例
例として以下を使った。

- **PA8** : 外付けLED出力
- **PA10** : 外付けボタン入力
- **USART2** : シリアル出力用

## 5.2 GPIO出力ピン設定
LED用のピンを **GPIO_Output** にする。

設定の意図：
- ソフトから `SET` / `RESET` を出したいから

## 5.3 GPIO入力ピン設定
ボタン用のピンを **GPIO_Input** にする。  
さらに **Pull-up** を有効にする。

設定の意図：
- 外付け抵抗なしで安定した入力にしたいから
- 押していないときは High
- 押すと Low

## 5.4 UART設定
UART送信用に USART2 を有効化する。

よく使う設定例：

- Baud Rate: `115200`
- Word Length: `8 Bits`
- Stop Bits: `1`
- Parity: `None`

---

## 6. CubeMXで生成される名前の意味

CubeMX でピンに名前を付けると、`main.h` にマクロ定義が生成される。

例：

```c
#define LED_EXT_Pin GPIO_PIN_8
#define LED_EXT_GPIO_Port GPIOA

#define BTN_EXT_Pin GPIO_PIN_10
#define BTN_EXT_GPIO_Port GPIOA
```

### これを使う理由
コード中で `GPIOA, GPIO_PIN_8` を直接書くより安全。

#### 直接書く場合
```c
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_SET);
```

#### CubeMXの名前を使う場合
```c
HAL_GPIO_WritePin(LED_EXT_GPIO_Port, LED_EXT_Pin, GPIO_PIN_SET);
```

### なぜこちらがよいか
後で CubeMX 側でピン変更しても、コード修正箇所が減るため。

---

## 7. HAL_GPIO の基本

HAL は、STM32の周辺機能を扱うためのライブラリ。

`HAL_GPIO_...` は GPIO 用の関数。

---

## 7.1 LEDを点灯/消灯する

### 点灯
```c
HAL_GPIO_WritePin(LED_EXT_GPIO_Port, LED_EXT_Pin, GPIO_PIN_SET);
```

### 消灯
```c
HAL_GPIO_WritePin(LED_EXT_GPIO_Port, LED_EXT_Pin, GPIO_PIN_RESET);
```

### 意味
- `GPIO_PIN_SET` : 出力を High にする
- `GPIO_PIN_RESET` : 出力を Low にする

### 注意
LEDの配線方法によっては High で点灯、Low で消灯とは限らない。  
今回は

- 出力ピン → 抵抗 → LED → GND

なので、
- High で点灯
- Low で消灯

---

## 7.2 ボタン状態を読む

```c
GPIO_PinState state = HAL_GPIO_ReadPin(BTN_EXT_GPIO_Port, BTN_EXT_Pin);
```

戻り値は `GPIO_PinState` 型で、主に以下の2つ。

- `GPIO_PIN_SET`
- `GPIO_PIN_RESET`

### Pull-up入力時の意味
- 押していない → `GPIO_PIN_SET`
- 押した → `GPIO_PIN_RESET`

---

## 7.3 ボタンでLEDを制御する例

```c
if (HAL_GPIO_ReadPin(BTN_EXT_GPIO_Port, BTN_EXT_Pin) == GPIO_PIN_RESET) {
    HAL_GPIO_WritePin(LED_EXT_GPIO_Port, LED_EXT_Pin, GPIO_PIN_SET);
} else {
    HAL_GPIO_WritePin(LED_EXT_GPIO_Port, LED_EXT_Pin, GPIO_PIN_RESET);
}
```

### このコードの意味
- ボタンが押されたらLEDを点灯
- 離したらLEDを消灯

---

## 8. UART の基本

UART は、シリアル通信を行うための仕組み。

今回の用途は、**PCへ文字列を送ること**。

---

## 8.1 文字列送信

```c
uint8_t msg[] = "Hello World !!!\r\n";
HAL_UART_Transmit(&huart2, msg, sizeof(msg) - 1, 100);
```

### 引数の意味
```c
HAL_UART_Transmit(ハンドル, 送信データ, データ長, タイムアウト);
```

#### `&huart2`
USART2 を操作するための情報

#### `msg`
送るデータ

#### `sizeof(msg) - 1`
文字列終端の `\0` を除いた長さ

#### `100`
タイムアウト時間

---

## 8.2 `\r\n` の意味
改行用。

- `\r` : キャリッジリターン
- `\n` : ラインフィード

シリアルターミナルでは `\r\n` を付けると見やすいことが多い。

---

## 8.3 UART使用時の注意
UART送信しても、PC側で以下ができていないと何も見えない。

- COMポートを開く
- ボーレートを一致させる（例: `115200`）
- ターミナルソフトを起動する

---

## 9. main.c の基本構造

STM32 の `main.c` は大まかに以下の流れ。

```c
int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_USART2_UART_Init();

    while (1)
    {
        // 繰り返したい処理
    }
}
```

### それぞれの役割

#### `HAL_Init();`
HALライブラリの初期化

#### `SystemClock_Config();`
クロック設定  
マイコン内部の動作周波数を決める

#### `MX_GPIO_Init();`
CubeMXが生成したGPIO初期化処理

#### `MX_USART2_UART_Init();`
CubeMXが生成したUSART2初期化処理

#### `while(1)`
無限ループ  
マイコンは普通、ここをずっと回り続ける

## 10. 関数を自分で分ける意味

処理を関数に分けると見やすくなる。

例：

```c
void btn_led_bread(void);
void btn_test(void);
void print_text(void);
```

### 分ける理由
- 役割ごとに整理できる
- バグを見つけやすい
- テストしやすい

---

## 11. ボタンの押下エッジ検出

「押している間ずっと実行」ではなく、  
**押した瞬間だけ1回実行したい** ときは、前回状態と今回状態を比べる。

### 例
```c
void btn_test(void)
{
    static GPIO_PinState pre_state = GPIO_PIN_SET;
    GPIO_PinState cur_state = HAL_GPIO_ReadPin(BTN_EXT_GPIO_Port, BTN_EXT_Pin);

    if (pre_state == GPIO_PIN_SET && cur_state == GPIO_PIN_RESET) {
        print_text();
    }

    pre_state = cur_state;
}
```

### この考え方
- 前回は押していなかった
- 今回は押された

この瞬間だけ `print_text()` を呼ぶ。

### `static` の意味
関数を抜けても値を保持したいから。  
前回状態を覚えておく必要があるため。

---

## 12. チャタリング

ボタンは、押した瞬間に内部接点が細かく揺れて、  
短時間に ON/OFF が何度も起きることがある。  
これを **チャタリング** という。

### 症状
1回押したのに、プログラム上では複数回押されたように見える。

### 簡易対策
```c
HAL_Delay(20);
```

を入れて少し待つ。

### 注意
これは簡易対策であり、本格的には

- 時間を使った判定
- タイマ利用
- 状態遷移での安定化処理

などがある。

---

## 13. `HAL_Delay()` の注意

`HAL_Delay(1000);` は 1000ms = 1秒待つ。

### 問題点
待っている間、他の処理が止まる。

例えば、

```c
for (int i = 0; i < 10; i++) {
    HAL_UART_Transmit(...);
    HAL_Delay(1000);
}
```

のようにすると、約10秒間メインループが止まる。

### 今回の学び
- 最初の動作確認には便利
- しかし多用すると「固まったように見える」

---

## 16. Nucleo のピンと STM32 の PA/PB 対応

**注意**  
Nucleo 基板の「D2」「D7」などは **Arduino互換ヘッダ名**。  
一方で STM32 内部の本当のピン名は `PA10` や `PA8` など。  
コードでは後者を扱う。

## 16.1 今回よく使った対応
Nucleo-F103RB では、今回の学習で主に以下を使った。

| Nucleo表記 | STM32側のピン名 | 用途例 |
|---|---|---|
| D7 | PA8 | 外付けLED出力 |
| D2 | PA10 | 外付けボタン入力 |
| D1 | PA2 | USART2 TX |
| D0 | PA3 | USART2 RX |

### 意味
- **D7** に線を挿したつもりでも、コード上では **PA8** を触っている
- **D2** に線を挿したつもりでも、コード上では **PA10** を読んでいる

---

## 16.2 USART2 とピン
USART2 はよく以下のピンに対応する。

| 機能 | STM32ピン |
|---|---|
| USART2_TX | PA2 |
| USART2_RX | PA3 |

### 注意
UARTを使う予定がある場合、PA2/PA3をGPIOとして雑に使わない方が混乱が少ない。

# タイマーまとめ

## 1. マイコンのタイマー基礎

### タイマーとは
タイマーは、**マイコンの中にある「時間を数えるための専用ハードウェア」**です。  
`for`文で無理やり待つのではなく、**内部のカウンタがクロックに合わせて数を数える**仕組みです。

---

### 何ができるか
タイマーを使うと、たとえば次のようなことができます。

- 1秒ごとにLEDを点滅する
- 10msごとにボタン状態を確認する
- 一定周期で割り込みを発生させる

---

### 基本用語

#### クロック
マイコンが動く基準の速さです。  
タイマーもこのクロックを元に数えます。

#### Prescaler（プリスケーラ）
**タイマーが数える速さを遅くする設定**です。  
速すぎるクロックを、そのまま使いやすい速さまで落とします。

#### ARR / Period
**どこまで数えたら1周期にするか**の設定です。  
この値まで数えたら、タイマーは更新イベントを起こします。

#### 割り込み
一定時間ごとに、**CPUへ「時間です」と知らせる仕組み**です。  
この通知を受けて、LED点滅などの処理を書きます。

---

### 周期の式
タイマー周期は基本的に次の式です。

```text
周期 = (Prescaler + 1) × (ARR + 1) / タイマクロック
```

つまり、

- Prescaler：数える速さを決める
- ARR：何回数えたら終わりにするか決める

の2段階で時間を作っています。

---

### 例
タイマクロックが 64MHz のとき、

- Prescaler = 60000
- ARR = 6000

なら

```text
周期 ≒ (60001 × 6001) / 64000000
    ≒ 5.63秒
```

となります。

---

## 2. CubeMX の設定

### 基本手順
1. **Timers** から使いたいタイマを選ぶ  
   例: `TIM2`

2. **Clock Source** を `Internal Clock` にする

3. **Parameter Settings** で次を設定する
   - Prescaler
   - Counter Period
   - Counter Mode は普通は `Up`

4. **NVIC Settings** で  
   `TIM2 global interrupt` にチェックを入れる

5. `.ioc` を保存して **Generate Code**

---

### よく見る設定項目

#### Clock Source = Internal Clock
マイコン内部クロックでタイマを動かす設定です。  
最初はこれで十分です。

#### Prescaler
タイマの数える速さを落とします。

#### Counter Period
何回数えたら更新イベントを出すか決めます。

#### NVIC
割り込みを有効にする設定です。  
これを入れないと、タイマが動いてもコールバック関数が呼ばれません。

---

### 1秒周期にしたい例
タイマクロックを 64MHz とすると、たとえば次のように考えます。

- Prescaler = 63999
- ARR = 999

すると

```text
64MHz / 64000 = 1000Hz
1000回で1秒
```

なので、約1秒ごとに割り込みできます。

---

## 3. main.c に書くこと

### 必要な流れ
`main.c` では主に次の2つを書きます。

1. タイマー開始
2. タイマー周期ごとの処理

---

### 3-1. タイマー開始
`main()` の初期化後にこれを書きます。

```c
HAL_TIM_Base_Start_IT(&htim2);
```

#### 意味
- `TIM2` を開始する
- `IT` は Interrupt の意味
- つまり **割り込み付きでタイマ開始**

---

### 3-2. 周期ごとの処理
タイマーが1周期終わるたびに、次の関数が呼ばれます。

```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM2)
    {
        HAL_GPIO_TogglePin(GREEN_LED_GPIO_Port, GREEN_LED_Pin);
        HAL_GPIO_TogglePin(RED_LED_GPIO_Port, RED_LED_Pin);
    }
}
```

#### 意味
- `TIM2` の割り込みだったら
- 緑LEDと赤LEDを反転する

---

## 4. サンプルソース

以下は、**TIM2のタイマー割り込みでLEDを一定周期で切り替える最小例**です。

```c
/* USER CODE BEGIN Includes */

/* USER CODE END Includes */

/* Private variables ---------------------------------------------------------*/
TIM_HandleTypeDef htim2;

/* USER CODE BEGIN PV */

/* USER CODE END PV */

int main(void)
{
  HAL_Init();
  SystemClock_Config();

  MX_GPIO_Init();
  MX_TIM2_Init();

  /* USER CODE BEGIN 2 */
  HAL_TIM_Base_Start_IT(&htim2);
  /* USER CODE END 2 */

  while (1)
  {
  }
}

/* USER CODE BEGIN 4 */
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM2)
    {
        HAL_GPIO_TogglePin(GREEN_LED_GPIO_Port, GREEN_LED_Pin);
        HAL_GPIO_TogglePin(RED_LED_GPIO_Port, RED_LED_Pin);
    }
}
/* USER CODE END 4 */
```

---

## 5. 重要な注意点

### 1. 関数名は完全一致
次は間違いです。

```c
HAL_TIM_PeriodElapsedCallBack
```

正しくはこれです。

```c
HAL_TIM_PeriodElapsedCallback
```

`B` が大文字ではなく、**Callback** で1語です。

---

### 2. 型名も完全一致
正しくはこれです。

```c
TIM_HandleTypeDef
```

間違えやすい例:

```c
TIM_HancleTypeDef
```

---

### 3. `Instance` は大文字
正しくはこれです。

```c
htim->Instance
```

C言語は大文字小文字を区別します。

---

### 4. NVIC を有効にする
CubeMXで `TIM2 global interrupt` を有効にしないと、  
`HAL_TIM_PeriodElapsedCallback()` は呼ばれません。

---

## 6. 最後に覚えること

タイマーはまず次の3つを覚えれば十分です。

- **タイマー = 時間を数える専用ハードウェア**
- **Prescaler = 数える速さを決める**
- **ARR = 何回数えたら1周期にするか決める**

そして実際には、

- CubeMXで TIM2 を有効化
- Prescaler と ARR を設定
- NVICで割り込み有効化
- `HAL_TIM_Base_Start_IT(&htim2);`
- `HAL_TIM_PeriodElapsedCallback()` に処理を書く

これが最初の基本形です。
